# Provider Submission Checklist

Before a provider plugin can be accepted into the registry:

- [ ] Includes a valid \`plugin.json\`.
- [ ] Uses a compatible license.
- [ ] Declares domains.
- [ ] Declares runtime requirements.
- [ ] Declares secrets.
- [ ] Declares filesystem/network permissions.
- [ ] Declares Bright Data policy.
- [ ] Includes sanitized fixtures.
- [ ] Includes parser tests.
- [ ] Includes health check command.
- [ ] Includes maintainer/support metadata.
- [ ] Contains no credentials.
- [ ] Contains no real bills.
- [ ] Contains no customer screenshots.
- [ ] Contains no session cookies.
